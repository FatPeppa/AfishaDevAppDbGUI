==========================
JAVA Директория. Структура
==========================
Директория common содержит стандартные классы и методы, используемые и наследуемые в реализации по предметной области:
    1. db. Содержит стандартные классы и методы для работы с БД
    2. properties. Содержит классы и перечисления с основными параметрами приложения. Также в данную директорию будет добавлен парсер
        файла настроек приложения
    3. sort. Содержит стандартный инструментарий сортировки
    4. validation. Содержит стандартные универсальные классы и методы для валидации на всех уровнях (слоях) приложения
Директория data содержит инструментарий для работы с БД (по предметной области), т.е. слой данных приложения:
    1. datasource. Содержит DAO интерфейсы и классы, сущности, а также класс базы данных
    2. repository. Содержит классы и интерфейсы репозиториев упрощения работы с БД
    3. validation. Содержит настройки системных проверок ФЛК по БД, а также реализацию самих проверок
Директория service содержит функциональные классы обработки, а также классы валидации слоя бизнес-логики приложения:
    1. logic. Содержит сервисы (функциональные классы обработки) слоя бизнес-логики приложения
    2. validation. Содержит классы валидации слоя бизнес-логики приложения
Директория controller содержит классы управления UI приложения и иную функциональность, соответствующие слою представления приложения (структура директории соответствует структуре контролируемых fxml-ресурсов)

===============================
Ресурсная директория. Структура
===============================
Корень директории содержит основные формы приложения (форма приветствия с выбором регистрации/аутентификации пользователем, форма аутентификации, форма регистрации, основная форма приложения).
Стоит отдельно подсветить, что основная форма приложения (main-view.fxml) является контейнером для таблиц и форм поиска и используется для взаимодействия с данными объектами соответственно.

Директория tables. Содержит формы таблиц для просмотра данных БД. Каждый из файлов данной директории в обязательном порядке содержит TableView.
При этом, функциональность взаимодействия с данными формами реализовывается либо полностью на стороне основной формы и её контроллера, либо
    распределенно на этих формах и осн. форме.

Директория filters. Содержит формы поиска данных (поля для ввода, соответствующие той или иной таблице). Функциональность взаимодействия
с данными формами реализовывается аналогично формам директории tables.

Директория baseviews. Содержит универсальные UI-компоненты приложения, такие как, например, формы сообщений об ошибках.


Группы кодов ошибок:
    1. 1000ХХХХ - Ошибки при работе с сущностями БД
    2. 1001ХХХХ - Ошибки при заполнении аргументов методов DAO
    3. 1002ХХХХ - Ошибки при работе с файлами в рамках сущностей БД (например, существование файла сборки по указанному пользователем пути)
    4. 1003ХХХХ - Бизнес-ошибки при регистрации и аутентификации пользователя
    5. 99999999 - Системная ошибка (используется, например, для оборачивания SQL-исключений в бизнесовую ошибку)
